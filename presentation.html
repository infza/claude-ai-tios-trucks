💡 Základné Tipy

Začni tu

1️⃣ Jasné a Konkrétne Otázky

❌ Zlé:

Pomôž mi s kódovaním.

✅ Dobré:

Pomôž mi napísať Python funkciu, ktorá čita CSV súbor a vracia priemernú hodnotu numerického stĺpca "price".

2️⃣ Kontextové Informácie

  • 🔧 Technologický stack
  • 📦 Verzia nástrojov
  • 🎯 Cieľ úlohy
  • 📤 Očakávaný výstup
  • ⚙️ Obmedzenia a požiadavky

3️⃣ Žiadajte Formátovanie

Určite, ako chceš výstup:

  • 📋 "Poskytnúť ako JSON"
  • 📊 "Formátovať ako tabuľka"
  • 📝 "Výstup ako Markdown"
  • 🔢 "Číselný zoznam"
  • 📑 "Polia s nadpismi"

4️⃣ Postupné Spresnenie

Prvá odpoveď nie je vždy dokonalá:

  • "Áno, ale pridaj viac príkladov"
  • "Zmeň to na Go namiesto Python"
  • "Zameraj sa viac na bezpečnosť"
  • "Urob to jednoduchšie"
  • "Pridaj komentáre do kódu"

5️⃣ Rozdeľ Komplexné Úlohy

❌ Zle:

Vytvor kompletný e-shop backend

✅ Dobre:

1. API endpoint autentifikácie 2. Endpointy pre produkty 3. Integrácia platieb

🎯 Pokročilé Triky

Staň sa expertem

1️⃣ Role-Based Prompting

Definuj rolu pre Claude:

Si skúsený Python vývojár 
s 10 rokmi skúsenosti.
Pomôž mi optimalizovať 
tento kód na výkon.

2️⃣ Extended Thinking

Pre zložité problémy:

Prosím, premysli si 
tento problém 
krok za krokom a potom 
mi poskytni kompletné 
riešenie.

3️⃣ Iteratívne Zlepšovanie

  • 📝 Počiatočný draft
  • 🔍 "Zameraj sa viac na X"
  • 📚 "Pridaj príklady"
  • ✂️ "Zjednoduš to"
  • 🎨 "Vylepši formátovanie"

4️⃣ Multi-Turn Konverzácie

Udržiavaj kontext v rámci rozhovoru:

1. Objasnite princípy X
2. Ako sa to vzťahuje na Y?
3. Môžeš mi to ukázať 
   na príklade?

5️⃣ Prompt Chaining

Spájaj viacero promptov v sérii:

  1. Vygeneruj dátové štruktúry
  2. Vytvor operácie s nimi
  3. Napíš testy
  4. Optimalizuj kód

6️⃣ Few-Shot Learning

Poskytnú príklady:

Príklad 1: Input → Output
Príklad 2: Input → Output
Príklad 3: Input → Output

Teraz pre moj prípad:
Input → ?

7️⃣ Chain-of-Thought Prompting

Požiadaj Claude, aby vysvetlil svoje myslenie:

  • ❓ Čo je problém?
  • 🤔 Aké sú možnosti?
  • ✅ Ktorá je najlepšia?
  • 💭 Prečo?

🎨 Umenie Promptovania

Vytvor dokonalý prompt

Štruktúra Efektívneho Prompta

[ROLA] - Kto si?
[KONTEXT] - Aká je situácia?
[ÚLOHA] - Čo máš robiť?
[FORMÁT] - Ako to chceš?
[PRÍKLADY] - Prípadne príklady
[OBMEDZENIA] - Aké sú limity?

Príklad Komplexného Prompta

Rola: Skúsený technický write 
s odbornými znalosťami cloudu

Kontext: Tímom sa ťažko pochopí, 
ako funguje Docker v produkcii.

Úloha: Napíš jasný, zjednodušený sprievodca.

Formát: Markdown s názvami, 
kódovými blokmi a tabuľkami

Publikum: Junior vývojári bez Docker skúsenosti

Dĺžka: 500-800 slov

Inklúzia: 3 praktické príklady

Bezpečné Slovníčky

Termín Efekt
"prosím" Zlepšuje tón
"krok za krokom" Podrobnejšie vysvetlenia
"ako začiatočník" Jednoduchšie vysvetlenia
"porovnaj" Viacero možností
"prečo" Hlbšie vysvetlenie

📊 Prípadové Štúdie

Praktické príklady

Prípadová Štúdia 1: Vývoj Aplikácie

Problém:

Potrebujem architektúru React aplikácie

Riešenie:

Navrhni architektúru React aplikácie pre e-shop.
Požiadavky:
- 50+ produktov
- Nákupný košík
- Viacerí používatelia
- SEO optimalizácia

Formát: Diagram + vysvetlenie + kódové príklady

Prípadová Štúdia 2: Ladenie Kódu

Problém:

Čudný bug v produkcii

Riešenie:

Analyzuj tento stack trace 
a pomôž mi nájsť bug.

[Stack trace]

Kontext:
- Node.js 18
- Express app
- PostgreSQL databáza
- Problém náhodný

Prípadová Štúdia 3: Dokumentácia

Problém:

Potrebujem API dokumentáciu

Riešenie:

Vytvor OpenAPI 3.0 špecifikáciu.

Endpointy:
- GET /users
- POST /users
- PUT /users/{id}
- DELETE /users/{id}

Formát: YAML s príkladmi

✅ Best Practices

Rób to správne

1️⃣ Bezpečnosť Promptov

✅ Robiť:

  • Anonymizovať dáta
  • Overovať citlivé výstupy
  • Rešpektovať súkromie

❌ Nerobiť:

  • Nedeliť API kľúče
  • Nedeliť hesla
  • Nedeliť obchodné tajomstvá

2️⃣ Efektivita

Typ Úlohy Odporúčaný Model
Rýchle otázky Sonnet alebo Haiku
Zložité úlohy Opus
Dlhý kontext Sonnet alebo Opus
Nákladnosť Zvážiť model vs. zložitosť

3️⃣ Kvalita Výstupu

  1. Jasne definuj očakávania
  2. Poskytnú kontext
  3. Požiadaj o vysvetlenia
  4. Iteruj na spätnej väzbe
  5. Overuj výstupy

4️⃣ Opakovateľnosť

  • 📝 Udržuj presné prompty
  • 📊 Zaznamenávaj efektívne techniky
  • 🔧 Vytvárač šablóny pre bežné úlohy
  • 📚 Zdieľaj poznatky s tímom

5️⃣ Čítanie Výstupu

  • 👁️ Čítaj pozorne
  • ✓ Overuj faktické tvrdenia
  • 🧪 Testuj kódové príklady
  • 🧠 Aplikuj kritické myslenie

🔒 Bezpečnosť

Chráň svoje dáta

Bezpečné Používanie Claude

✅ Robiť:

  • Nedeliť citlivé informácie
  • Anonymizovať osobné údaje
  • Overovať citlivé výstupy
  • Používať bezpečné kanály
  • Dodržiavať politiku podniku

Čo Nerobiť

  • ❌ Nedeliť hesla alebo kľúče
  • ❌ Nedeliť osobné údaje
  • ❌ Nedeliť obchodné tajomstvá
  • ❌ Veriť slepě bez overenia
  • ❌ Ignorovať bezpečnostné varovania

Verifikácia Výstupu

  1. 🧪 Testuj kódové príklady
  2. ✓ Overi faktické tvrdenia
  3. 🔍 Kontroluj bezpečnostné obavy
  4. ⚠️ Validuj citlivé výstupy
  5. 👥 Konzultuj s expertmi ak je potrebné

🚀 Pokročilé Techniky

Pre profesionálov

1️⃣ Rubber Duck Debugging

Prompt: Pomôž mi debugovať 
tento kód ako "rubber duck".
Pýtaj sa ma otázky, 
ktoré ma navnú na problém.

[KÓD]

2️⃣ Analogické Myslenie

Vysvetli [KOMPLEXNÝ KONCEPT] 
pomocou analógie s 
[JEDNODUCHÉ VECI].

3️⃣ Brainstorming

Generuj [N] inovatívnych 
nápadov na [PROBLÉM].

Každá ideá by mala byť 
jedinečná a praktická.

4️⃣ Analýza Konkurencie

Porovnaj [PRODUKT A] 
s [PRODUKTOM B].

Tabulka výhod, nevýhod, 
ceny, funkcií.
